## Comprendre

Ce projet démarre d’un `Create Next App` / `Shadcn` / `Drizzle` / `NextAuth` avec la même architecture que le module `Clean Architecture`

```bash
src/       # Tests unitaires et d'intégration
├── app/            # Pages, API et layouts (App Router)
├── components/     # Composants réutilisables
├── db/             # Configuration et interactions avec la base de donné
├── lib/            # Bibliothèques et fonctions utilitaires
├── services/       # Logique métier (API, auth, etc.)
└── types/          # Définitions des types TypeScript
```

📑 Le liens vers la doc [https://fr.reactjs.org/docs/getting-started.html](https://fr.reactjs.org/docs/getting-started.html)

## Exercice

- **🐶** Dans un premier temps tu vas devoir créer une base de données `Postgres` sur `Vercel` et coller

```bash
#.env.development
DATABASE_URL="postgres://default:xxxxx@hostname.tech:5432/verceldb?sslmode=require"
```

Exécute les scripts de manipulation de BDD

```bash
pnpm db:generate
pnpm db:migrate
pnpm db:check
pnpm db:reset-seed
```

Fichiers

- `.env.development`
- `package.json`

## Bonus

### 1. 🚀 Générer la page d’accueil

La `homepage` est une des pages les plus importantes d’un SaaS est mériterais une formation à part entière. Ce n’est pas le but de cette formation. Pour simplifier tu peux

- **🐶** Demande a V0 de `Vercel` ou autre IA de te générer une `homepage` pour ton SaaS.
- **🐶** Ou utilise celle dans `src/app/page.final`

Fichiers

- `src/app/page.final`
- `src/app/page`

### 2. 🚀 Vérification des routes

Pour le moment tout est publique mais par la suite l’application sera découper en 3/

- Toutes les routes publiques
- Toutes les routes lié à l’authentification
- Toutes les routes lié à l’application connecté

```bash
src/                 # Tests unitaires et d'intégration
├── app/             # Pages, API et layouts (App Router)
│   ├── (public)/      # Pages accessibles publiquement (landing page, contact, etc.)
│   │   ├── layout.tsx
│   │   ├── page.tsx
│   │   └── terms/
│   │       └── page.tsx
│   ├── (auth)/        # Pages pour l'authentification (login, signup, etc.)
│   │   ├── layout.tsx
│   │   ├── sign-in/
│   │   │   └── page.tsx
│   │   └── sign-up/
│   │       └── page.tsx
│   ├── (app)/         # Pages protégées pour les utilisateurs authentifiés
│       ├── layout.tsx
│       ├── dashboard/
│       │   └── page.tsx
│       └── account/
│           └── page.tsx
```

Penser a avoir `AUTH_SECRET` avec par exemple `openssl rand -hex 32`

```bash
#.env.development
AUTH_SECRET="xxx"
```

- Vérifie la présence de
  - [http://localhost:3000/sign-in](http://localhost:3000/sign-in)
  - [http://localhost:3000/sign-up](http://localhost:3000/sign-up)
  - [http://localhost:3000/privacy](http://localhost:3000/privacy)
  - [http://localhost:3000/terms](http://localhost:3000/terms)
  - [http://localhost:3000/dashboard](http://localhost:3000/dashboard)
