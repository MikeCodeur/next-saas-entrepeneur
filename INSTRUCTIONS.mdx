## Comprendre

Dans ce module, nous allons explorer **`React Email`**, une bibliothèque puissante pour créer des emails dynamiques et réutilisables avec `React`.

Associée à **`Resend`**, une solution moderne pour l'envoi d'emails, nous apprendrons à concevoir et envoyer des emails professionnels facilement, tout en intégrant des pratiques modernes pour améliorer l'expérience utilisateur.

Le développement y est simplifié car `react-email` inclut un serveur de visualisation

## Bonus

### 1. 🚀 Email Service

L’envoi d’email dans une application est essentiel. Il peut y en avoir pour les utilisateurs pour les connexions etc… mais aussi pour les administrateurs pour être notifiés de ce qu’il se passe, comme par exemple la création d’un nouvel utilisateur.

Pour simplifier l’exercice, nous avons créé le service email qui contient 2 fonctions :

- `SendEmail` (fonction que l’on pourra utiliser dans `auth-config` par exemple)
- `sendInternalEmail` : fonction que l’on va utiliser sur `sign-up`

```
export const sendEmail = async (
  payload: CreateEmailOptions,
  options?: CreateEmailRequestOptions
) => {
  if (process.env.NODE_ENV === 'development') {
    payload.subject = `[DEV] ${payload.subject}`
  }

  const {error} = await resend.emails.send(
    {
      ...payload,
      from: 'onboarding@resend.dev',
      to: 'delivered@resend.dev',
    },
    options
  )

  if (error) {
    console.error(error)
    throw error
  }
}
```

- **🐶** Utilise ces 2 fonctions : Dans `auth-config` et `sign-up`

Fichiers

- `services/authentication/auth-config.ts`
- `services/email-service.ts`
- `app/(auth)/sign-up/action.ts`
